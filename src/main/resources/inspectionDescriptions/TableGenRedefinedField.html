Reports fields that redefine a field already declared by the enclosing class, definition or one of its base classes.

Although legal TableGen, redeclaring an inherited field merely to give it a new value shadows the original declaration and is a code smell. A let statement expresses the intent of overriding an inherited field's value without redeclaring it.

Example:


class Base {
  int x = 0;
}

class Derived : Base {
  int x = 1; // redefinition of 'x'
}

After applying the quick fix:


class Derived : Base {
  let x = 1;
}
